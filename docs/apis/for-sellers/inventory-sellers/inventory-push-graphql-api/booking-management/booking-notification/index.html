Booking Notification

During the booking process, we'll send you a BookNotif request message to share reservation details. You just need to check and let us know the reservation status. After that, Travelgate will handle the response from your system, moving forward with the reservation and updating you on the status.

tip

If you have the booking locator, include it in the BookNotif response, this way we'll include it in our response.

Keep in mind that if there's an issue with the reservation in your system, we'll consider it as not successful (NOK) and we return an error message.

Request Headers​

Our system sends BookNotif messages as POST request with the following headers.

Content Type: Content-Type header provides information about the actual content type of the returned content. In this case, the header is set to:

Content-Type: application/json;charset=UTF-8

Authentication:

Authorization: Basic xxxxxxxxxxxxxxxxx
note

You need to provide an endpoint to which we will send the bookNotif and a username and password.

Request Data Breakdown​

bookNotif

Root object containing all booking information.

settings

Settings for the booking notification:

  • supplier (string): Your inventory supplier identifier.
  • client (string): The inventory client identifier.
bookDetails

Main details of the booking:

  • status (BookStatus): Status of the booking. Possible values are:
    • Confirmed
    • Cancelled
  • references (array of Reference):
    Reference Object
    • locator (string): Unique identifier for the reference.
    • type (ReferenceType): Type of reference. Possible values are:
      • Client
      • Internal
  • holder (Customer):
    Customer Object
    • name (string): First name of the holder.
    • surname (string): Last name of the holder.
    • personalTitleType (PersonalTitleType): Title of the holder. Possible values are:
      • MR
      • MRS
      • MISS
      • MS
    • age (integer, optional): Age of the holder.
    • contactInfo (object, optional):
      ContactInfo Object
      • email (string, optional): Email address of the holder.
      • phone (string, optional): Phone number of the holder.
  • hotel (HotelDetail):
    HotelDetail Object
    • bookingDate (DateTime): Date when the booking was created.
    • lastModifyDate (DateTime, optional): Date when the booking was last modified.
    • checkIn (DateTime): Check-in date.
    • checkOut (DateTime): Check-out date.
    • hotelCode (string): Unique identifier for the hotel.
    • context (string): Context for the hotel, usually TGX_PUSH. If you need to use your own hotel codes, please contact with us and we create your own context code.
    • hotelName (string): Name of the hotel.
    • boardCode (string): Code of the board.
    • boardName (string): Name of the board.
      List of possible board names with their corresponding codes:
      • 1: All_inclusive
      • 2: American
      • 3: Bed_And_breakfast
      • 4: Buffet_And_breakfast
      • 5: Caribbean_breakfast
      • 6: Continental_breakfast
      • 7: English_breakfast
      • 8: European_plan
      • 9: Family_plan
      • 10: Full_board
      • 11: Full_breakfast
      • 12: Half_board
      • 13: As_brochured
      • 14: Room_only
      • 15: Self_catering
      • 16: Bermuda
      • 17: Dinner_And_breakfast
      • 18: Family_American
      • 19: Breakfast

        RatePlan Object

        The RatePlan object provides detailed information about the pricing plan associated with a room or booking. This includes specific rules, payment options, and additional details about the plan.

        • name (string, optional): A descriptive name for the rate plan (e.g., "Standard Rate" or "Non-Refundable").

          • cardTypes (array of CreditCardType, optional): Specifies the types of credit cards accepted (e.g., VI for Visa, CA for Mastercard). See the possible values here

        • additionalDetails (AdditionalDetails, optional): A dictionary of additional details related to the rate plan. Each key is an AdditionalDetailType, and the value is a list of AdditionalDetail objects. Common detail types include:

          • Others (Not used)

          • SpecificClients

            AdditionalDetail Object

            The AdditionalDetail object contains extra information about the rate plan:

            • code (string): A unique identifier for the detail.
            • description (string): A textual description of the additional detail.

        CancelPolicy Object
        • refundable (boolean): Indicates if the room is refundable or not.
        • cancelPenalties (array of CancelPenalty): List of penalties that apply when the booking is canceled.
          CancelPenalty Object
          • hoursBefore (integer): The number of hours before the booking's check-in date when this penalty applies. For example, a penalty with hoursBefore: 72 applies if the booking is canceled less than 72 hours before check-in.
          • penaltyType (PenaltyType): The type of penalty applied. Possible values include:
            • Amount: A fixed monetary penalty.
            • Nights: A penalty based on the price of one or more nights.
            • Percentage: A percentage of the total booking price.
          • currency (string, required for Amount): The currency in which the penalty is calculated, specified in ISO 4217 format (e.g., "USD", "EUR"). Required if penaltyType is Amount.
          • value (number): The value of the penalty. Its meaning depends on the penaltyType:
            • If penaltyType is Amount, this is the fixed monetary value (e.g., 100.0).
            • If penaltyType is Nights, this represents the number of nights charged as a penalty.
            • If penaltyType is Percentage, this is the percentage of the total booking price (e.g., 25.0 for 25%).
          • deadline (string, optional): A specific date and time (in ISO 8601 format) when this penalty takes effect. This is often used in addition to hoursBefore for clarity.

      • remarks (array of Remark, optional): List of remarks or special instructions for the room.

        Remark Object
        • text (string): The remark or note related to the room.
  • price (Price): Total price of the booking.
    Price Object. More information here
    • currency (string): The currency in which the price is expressed, using the ISO 4217 standard (e.g., "USD" for US dollars, "EUR" for euros).
    • binding (boolean): Indicates whether the price is binding (true) or not (false). A binding price means that the buyer must sell the product at least at the minimumSellingPrice or the gross price, depending on what the seller requires. If binding is false, the buyer has more flexibility to set the selling price.
    • net (number): Represents the net amount of the price before adding any commission, taxes, or other charges. It is the base cost of the product that the buyer pays to the seller.
    • gross (number): The gross amount of the price, including commissions, taxes, or other applicable fees. This represents the total cost of the product without discounts.
    • minimumSellingPrice (number): Defines the minimum allowable selling price for the product. It serves as a restriction to ensure the buyer does not sell the product below this amount. This field is particularly useful for protecting the sellers's pricing policies and ensuring consistent pricing across sellers.

BookNotif Response​

After each request, a response should be provided from yours side. If a bookNotif request is received, a corresponding bookNotifRs response should be sent. There are two response options, success or error.

  • Success
  • Error
{
    "bookNotifRs":{
        "dateTime": "2025-01-15T14:10:07.878425Z",
        "status": "Confirmed",
        "references":[
            {
                "locator": "your_locator",
                "type": "Provider"
            }
        ]
    }
}
{
    "bookNotifRs":{
        "dateTime": "2025-01-15T14:10:07.878425Z",
        "adviseMessages":[
            {
                "code":"10",
                "description":"HotelCode not found"
            }
        ]
    }
}

Response Data Breakdown​

bookNotifRs

Root object containing the response information.

  • dateTime (DateTime): Date time when the booking notification was received.
  • status (BookStatus, optional): Status of the booking, mandatory if success response. Possible value are:
    • Confirmed: when bookNotif has bookDetails with status Confirmed
    • Cancelled: when bookNotif has bookDetails with status Cancelled
  • references (array of one Reference, optional):
    Reference Object
    • locator (string): Unique identifier for the reference.
    • type (ReferenceType): Type of reference. Possible values is Provider
  • adviseMessages (array of AdviseMessage, optional): Mandatory if error response.
    AdviseMessage Object
    • code: Possible values are:
      • -1: Unexpected error
      • 1: Validation error
      • 10: HotelCode not found
      • 11: Invalid dates
      • 14: Invalid derived rate
      • 22: Rooms not found
      • 23: Rates not found
      • 30: Occupancy error
      • 31: RatePlan rate error
      • 38: Credetials not found
      • 50: Confirmation error
      • 60: Cancellation error
    • description (string): Description of the error.
